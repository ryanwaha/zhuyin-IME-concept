# 注音輸入法設計原型
**版本 0.4 — 雙字元 Lookahead 與 ε-transition 上限**

---

## 一、核心設計意圖

> 無論輸入序列是否完全合法，系統都應試圖提出一個最合理的國字序列。

系統的核心任務是從**帶噪的擊鍵序列**中，逆向推斷使用者的**語言意圖**。

---

## 二、基本假設

**使用者能力預設**
使用者的注音能力預設為正常。偏差來源為**手指的機械失誤**。

**準確性優先**
系統不設硬性效能上限。連續錯字導致的計算延遲是自然懲罰，不以人為截斷換取流暢。

**音韻合法性的角色**
音韻合法性不作為過濾器，不剪枝。音節槽位結構 `[C]?[M]?[F]?T` 作為**隱藏生成模型**，編入 DP 的狀態空間與代價函數，而非硬性約束。

---

## 三、整體架構

```
帶噪 keystroke 流（使用者輸入）
        │
        ▼
  ┌─────────────┐
  │  前處理層   │  T 鍵錨點偵測 → 區間切割 → 異常區間修正
  └─────────────┘
        │ keystroke 區間序列
        ▼
  ┌─────────────┐
  │   第一段    │  區間內 DP → 候選音節集 → 建構 Lattice
  │  注音還原   │
  └─────────────┘
        │
        │  軟介面：Lattice（攜帶路徑信心度的 DAG）
        │
        ▼
  ┌─────────────┐
  │   第二段    │  NLP 問題（Viterbi Search）
  │  注音選字   │  ※ 暫不在 POC scope
  └─────────────┘
        │
        ▼
   國字序列（輸出）
```

## 四、前處理層

### 4.1 職責

判斷 keystroke 區間在槽位結構上是否構成一個完整音節。
判斷標準純粹基於 `[C]?[M]?[F]?T` 的符號集查表，不涉及音韻合法性，為 O(1) 操作。

### 4.2 T 鍵錨點

聲調鍵（`space`、`3`、`4`、`6`、`7`）的鍵集與聲母韻母幾乎不重疊，偵測信心度天然高。
每個 T 鍵作為音節末尾錨點，進行初始區間切割。

### 4.3 區間合法性驗證

```
初始區間切割（T 鍵）
        ↓
逐區間合法性判斷
        ↓
連續兩組合法排列？
  YES → 強錨點，邊界確定，獨立 DP
  NO  → 弱錨點，合併相鄰區間，擴窗 DP
```

連續兩組合法音節同時出現作為強錨點，將相位漂移的損害嚴格限制在區間內。

### 4.4 異常區間的信號

| 區間長度 | 診斷 | 處理 |
|---|---|---|
| > 4 | T 被刪除，兩音節合併 | 與相鄰區間合併擴窗 |
| < 2 | 假錨點，誤判 T | 與相鄰區間合併 |
| 2–4 | 正常 | 獨立 DP |

**T 的刪除由前處理層的區間合併機制處理，不進入區間內 DP 的責任範圍。**
這保證 DP 的終止條件永遠是確定的。

## 五、DP 形式化定義

### 5.1 輸入

$$I = (k_1, k_2, \dots, k_n)$$

由前處理層切割出的 keystroke 區間，其中 $k_n$ 為 T 錨點。

### 5.2 狀態空間

$$S = \{C, M, F, T\} \times \{0, 1\} \cup \{\text{NULL}\}$$

- $\{C,M,F,T\} \times \{0,1\}$：槽位 × 是否填充（對應 `[C]?[M]?[F]?` 的可選性）
- $\text{NULL}$：keystroke 為贅餘插入，不對應任何槽位

### 5.3 轉移函數

**單鍵轉移：**

$$\delta(k_i, s) = \text{cost}(k_i \to s) \quad s \in \{C, M, F, T\}$$

$$\delta(k_i, \text{NULL}) = p_{\text{ins}} \quad \text{（插入懲罰）}$$

$$\delta(\varepsilon, s) = p_{\text{del}} \quad \text{（ε-transition，刪除懲罰，不消耗 keystroke）}$$

**雙字元 Lookahead（換位）：**

$$\delta(k_i, k_{i+1}, s_a, s_b) = p_{\text{trans}} \quad \text{其中 } s_a \prec s_b$$

當相鄰兩個 keystroke 的最佳槽位分配與其出現順序相反時，識別為單一換位事件，代價為 $p_{\text{trans}}$，而非兩次獨立替換。

$$p_{\text{trans}} \ll 2 \times p_{\text{sub}}$$

確保換位被識別為低代價的單一事件，而非兩個高代價替換的累加。

**大小關係：** $p_{\text{del}} < p_{\text{ins}}$（漏打比多打更常見）

POC 階段，所有代價以 magic number 實作，訓練後由學習到的權重替換。

### 5.4 偏序約束

$$s_i \preceq s_j \quad \forall s_i, s_j \notin \{\text{NULL}\},\; i < j$$

其中 $\preceq$ 為 $C \to M \to F \to T$ 的嚴格偏序。NULL 在偏序中透明，不打斷槽位順序。

### 5.5 ε-transition 的作用域與觸發上限

**作用域：**

$$\varepsilon\text{-transition 作用於 } \{C, M, F\}$$

T 的刪除由前處理層處理，不在此範圍內。

**觸發上限（防止 DP 退化）：**

不設上限時，DP 可透過連續刪除「解釋」任意序列，使代價函數失去區分力。以下兩個約束共同防止退化：

*連續觸發上限：*

$$\text{連續 ε-transition} \leq 1$$

不允許兩個相鄰槽位同時被刪除。

*單區間觸發上限：*

$$\text{單區間 ε-transition 總數} \leq \left\lfloor \frac{n}{2} \right\rfloor$$

其中 $n$ 為區間長度。區間越短，允許的刪除次數越少，防止短區間被過度解構。

### 5.6 輸出

$$\text{DP}(I) = \{(\sigma, w)\}$$

候選音節集合，每個元素為音節符號序列 $\sigma$ 與對應總代價 $w$。

## 六、Lattice 形式化定義

### 6.1 形式

$$\mathcal{L} = (V, E, W)$$

**節點集合：**
$$V = \{v_0, v_1, \dots, v_m, v_{\text{end}}\}$$

節點 = keystroke 位置（確定值），加上起點 $v_0$ 與終點 $v_{\text{end}}$。

**邊集合：**
$$E \subseteq V \times V$$

**權重函數：**
$$W: E \to [0, 1]$$

### 6.2 邊的語意

$$W(v_i, v_j) = \text{confidence}(\sigma_{i \to j})$$

從 $v_i$ 到 $v_j$ 對應一個候選音節 $\sigma$，信心度由 DP 輸出代價轉換：

$$W = e^{-w}$$

代價越低，信心度越高。

### 6.3 邊的類型

| 邊類型 | 語意 | 權重來源 |
|---|---|---|
| 正常邊 | keystroke → 槽位 | 噪音模型代價 |
| NULL 邊 | keystroke → 被忽略（插入錯誤） | $p_{\text{ins}}$ |
| ε 邊 | 槽位被跳過（刪除錯誤） | $p_{\text{del}}$ |

### 6.4 路徑的語意

$$\text{path} = v_0 \to \cdots \to v_{\text{end}}$$

一條完整路徑 = 對整個輸入序列的**一種音節切分與符號還原假設**。

### 6.5 DP 到 Lattice 的介面

$$\text{DP}(I) \xrightarrow{\text{建圖}} \mathcal{L}$$

DP 的每個輸出 $(\sigma, w)$ 對應 Lattice 中的一條邊。

## 七、三軸噪音模型

第一段在還原注音時，依據三個維度計算 DP 代價：

### 第一軸：符號類型異質性
C / M / F / T 各有不同的基礎替換率（C 最大、M 最小）。

### 第二軸：鍵位方向異質性
- **垂直鄰近**：同列異行，語音相近，權重最高
- **水平鄰近**：同行異列，權重次之
- **對角鄰近**：跨行跨列，物理機率低，權重最低

### 第三軸：失誤模式

| 模式 | POC 狀態 | 處理位置 |
|---|---|---|
| 替換（Substitution） | ✅ In scope | DP 單鍵轉移 |
| 換位（Transposition） | ✅ In scope | DP 雙字元 Lookahead |
| 刪除（Deletion） | ✅ In scope（區間內） | ε-transition（含上限） |
| 插入（Insertion） | ✅ In scope（區間內） | NULL 狀態 |
| T 的刪除 | ⚠️ 前處理層處理 | 區間合併 |

## 八、偏置矩陣

### 定義

$$B[k] = \{k_1: p_1, k_2: p_2, \dots\}$$

編碼「observed keystroke $k$ 實際上是其他鍵」的機率分布，來源為：
- **鍵盤幾何**：物理鄰近距離
- **常見誤用**：如 ㄣ↔ㄥ、ㄧ↔ㄩ

### 架構定位

偏置矩陣為**訓練時工具，不出現在運行時架構中**：

| 職責 | 去向 |
|---|---|
| 鍵位幾何編碼 | 訓練資料生成管線 |
| 常見誤用編碼 | 訓練資料生成管線 |
| 運行時查表 | 消除 |

模型從帶偏置的訓練樣本中隱式學到這些分布，運行時不需額外查表。

---

## 九、訓練資料生成管線

1. **文本語料庫**：提取乾淨國字序列
2. **音節轉換**：將國字轉為標準 keystroke 序列
3. **三軸噪聲注入**（以偏置矩陣指導分布）：
   - 根據符號類型（C/M/F/T）決定基礎機率
   - 根據方向異質性（垂直/水平/對角）挑選替換目標
   - 組合替換、換位、刪除、插入操作
4. **樣本生成**：輸出帶物理機率標記的 (clean, noisy) 訓練對
